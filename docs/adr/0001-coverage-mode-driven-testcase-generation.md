# 采用覆盖模式驱动的测试用例生成策略

用例生成工具的目标是生成尽量接近测试专家设计结果的测试用例，因此不再把固定生成条数作为主要质量目标，而是采用快速覆盖、标准覆盖、专家覆盖三档覆盖模式，并使用最大条数上限控制规模。快速覆盖优先速度，标准覆盖先做内部需求分析再生成用例，专家覆盖在分析和生成后再做覆盖自检补全；后端内部执行质量校验和一次自动修复，前端不展示分析过程或质量提示，只展示生成任务和有效用例结果。

## Consequences

- 最大条数上限是硬上限，AI 可以少于上限，但不能为了凑满数量牺牲用例质量。
- 用例编号由后端统一重排，功能测试使用 `TC001...`，API 测试使用 `API-TC001...`。
- 覆盖维度不新增为表格列，应体现在“功能测试点”或“用例标题”中，以保持现有 8 列格式和导出格式稳定。
